# AGENTS Notes

## Startup

- Read `CLOUD.md` at the beginning of work in this repository.
- If `CLOUD.md` is missing, continue with best effort and mention it briefly.

## Project Context

- This is a small project maintained and used by one person.
- Prefer practical, simple solutions over "enterprise" architecture.
- Avoid overengineering and unnecessary abstractions.
- Keep implementation and maintenance cost low.

## Engineering Priorities

- Optimize for speed of iteration, clarity, and reliability.
- Keep backward compatibility where cheap; do not add complexity only for theoretical future needs.
- Prefer small, targeted tests and smoke checks over large testing frameworks unless explicitly requested.
- Introduce extra layers, specs, and automation only when they bring clear value now.
